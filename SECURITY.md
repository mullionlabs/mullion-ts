# Security Policy

## Reporting a vulnerability

If you believe you’ve found a security vulnerability in Mullion, please **do not** open a public GitHub issue.

Instead, use **GitHub Security Advisories** for this repository to report it privately.

What to include:

- A clear description of the vulnerability and impact
- Reproduction steps (minimal code sample if possible)
- Affected package(s) and versions
- Any suggested mitigation or patch

We’ll respond as quickly as possible and coordinate a fix + disclosure.

## Supported versions

Mullion is currently under active development (pre-1.0).  
We generally support the latest published versions of the packages in this monorepo.
